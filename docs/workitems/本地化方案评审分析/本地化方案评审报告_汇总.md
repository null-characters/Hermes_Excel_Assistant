## 四、已确认决策

> **决策日期**: 2026-05-15
> **决策依据**: 双视角评审 + 用户确认

### 4.1 方向性决策（已确认）

| 决策项 | 选择 | 原因 |
|--------|------|------|
| **D1: 技术路线** | ✅ 继续本地化 | Hermes + 自定义 LLM 已验证可行 |
| **D2: 产品化策略** | ✅ 并行推进 | 安全修复 + Web UI 同步开发，效率最高 |
| **D3: Excel 存储** | ✅ 本地文件系统 | 简化架构，避免 MinIO 链路断裂 |
| **D4: 沙箱方案** | ✅ local + SQLite 隔离 | 数据隔离 + 路径白名单，无 Docker 沙箱 |
| **D5: LLM 配置** | ✅ 用户自定义 | 用户自行配置 API Key 和 Provider |

### 4.2 技术选型（已确认）

| 场景 | 选择 | 理由 |
|------|------|------|
| Web UI | Streamlit | 快速原型，Python 原生，4h 可完成 |
| 数据隔离 | SQLite 多文件 | 会话级隔离，清理简单，无需额外依赖 |
| 文件存储 | 本地文件系统 | 简化架构，避免 MinIO 依赖 |
| LLM 配置 | 用户自定义 | 用户在 `.env` 或 Web UI 设置页面配置 |
| Docker SDK | docker-py（后续） | 官方库，避免 subprocess 注入 |
| 任务队列 | ARQ（后续） | 轻量，无需额外 Redis |
