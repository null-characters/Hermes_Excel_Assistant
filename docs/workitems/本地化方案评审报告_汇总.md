### 1.2 综合判定

> **当前方案是成功的 PoC（概念验证），但不是可交付的 MVP。**

```
PoC 验证结果：
  ✅ Hermes Agent + GLM-5 可以处理 Excel 任务
  ✅ 本地化部署方案可行（降低企微集成门槛）
  ✅ REST API 桥接模式可行

MVP 缺口：
  ❌ 目标用户（文职人员）无法使用 CLI 交互
  ❌ Excel 处理链路断裂（文件无法传递到 Agent）
  ❌ 安全问题导致生产环境不可部署
  ❌ 无测试覆盖，质量无保障
```

## 四、决策记录

### 4.1 方向性决策

| 决策 | 内容 | 原因 |
|------|------|------|
| **D1: 优先 Web UI** | 先开发 Streamlit 界面，暂缓企微集成 | 回归产品定位，降低验证门槛 |
| **D2: 修复现有方案** | 不重构架构，在现有代码上修补 | 时间成本最小化 |
| **D3: 保留 local 后端** | 短期接受 local 终端风险，优先产品化 | 先验证用户价值 |

### 4.2 技术选型建议

| 场景 | 建议 | 理由 |
|------|------|------|
| Web UI | Streamlit | 快速原型，Python 原生，4h 可完成 |
| 任务队列 | ARQ (asyncio) | 轻量，无需额外 Redis |
| 流式输出 | SSE (Server-Sent Events) | HTTP 原生支持，兼容性好 |
| Docker SDK | docker-py | 官方库，避免 subprocess 注入 |
