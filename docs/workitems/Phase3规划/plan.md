## 三、方案设计

### 3.1 B-01 解决方案

**方案选择**：限制 Docker exec 目标容器

```python
# hermes_client.py 修改
ALLOWED_CONTAINERS = {"hermes-agent"}

def execute_task(self, prompt: str, session_id: str) -> dict:
    # 验证容器名称
    if self.container_name not in ALLOWED_CONTAINERS:
        raise SecurityError(f"Container {self.container_name} not allowed")
    # ... 原有逻辑
```

**优势**：
- 改动最小（仅修改 `hermes_client.py`）
- 向后兼容（不影响现有 API）
- 安全可控（白名单机制）

### 3.2 功能增强设计

#### 批量处理

```
用户上传多文件 → Web UI 显示列表 → 逐个/并行处理 → 结果打包下载
```

#### 处理模板保存

```
用户保存指令为模板 → 存储到 SQLite → Web UI 模板列表 → 一键复用
```

#### 结果预览

```
处理完成 → Web UI 内嵌预览 → 支持表格/图表渲染 → 无需下载即可查看
```

### 3.3 架构调整

```
Phase 3 架构（本地化增强）
│
├── Gateway: nginx-proxy (HTTPS 终止)
│
├── Services:
│   ├── hermes-bridge (API + Docker exec，受限)
│   ├── session_manager (会话管理)
│   └── web-ui (Streamlit，增强)
│
├── Agent: hermes-agent (LLM + 代码执行)
│
└── Storage: 本地文件系统 + SQLite
    ├── data/sessions/{session_id}/
    │   ├── uploads/
    │   ├── results/
    │   └── session.db
    └── templates/  (新增：处理模板)
```

## 四、风险点与避坑说明

### 4.1 风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| B-01 修复影响现有功能 | 低 | 高 | 充分测试 + 回滚方案 |
| 批量处理内存溢出 | 中 | 中 | 文件数量限制 + 流式处理 |
| 模板存储 SQL 注入 | 低 | 高 | 参数化查询 |
| 预览组件兼容性 | 中 | 低 | 支持 xlsx/csv，其他格式下载查看 |

### 4.2 避坑指南

1. **B-01 修复**：先在测试环境验证，确认 hermes-agent 可正常执行
2. **批量处理**：限制单次最多 10 个文件，避免资源耗尽
3. **模板保存**：使用 session_manager 现有存储层，不新增依赖
