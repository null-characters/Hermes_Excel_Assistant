## 检查点机制

### CP1: 安全修复验收 (Day 1 结束)

**触发条件**: T03-01 ~ T03-04 完成

| 检查项 | 验收标准 | 方式 |
|--------|----------|------|
| exec 白名单 | Bridge 仅可 exec hermes-agent | 单元测试 |
| 回归测试 | 现有功能无影响 | E2E 测试 |
| Docker 权限 | 无不必要权限 | docker inspect |

### CP2: 功能增强验收 (Day 3 结束)

**触发条件**: T03-05 ~ T03-19 完成

| 检查项 | 验收标准 | 方式 |
|--------|----------|------|
| 批量上传 | 5 文件同时上传处理 | 手动测试 |
| 模板保存 | 保存/加载/执行模板 | 手动测试 |
| 结果预览 | 表格正确渲染 | 手动测试 |

### CP3: 质量提升 + Phase 3 结项 (Day 5 结束)

**触发条件**: T03-20 ~ T03-23 完成 + 全量回归

| 检查项 | 验收标准 | 方式 |
|--------|----------|------|
| 架构文档 | 反映当前实现 | 文档审查 |
| 测试覆盖 | Bridge >80% | pytest --cov |
| CORS 配置 | 环境变量控制 | 配置验证 |
| 文件限制 | 大小限制生效 | 边界测试 |
