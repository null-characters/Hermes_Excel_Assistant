```

```

---

### CP1: 安全修复验收 (Day 1 结束)

**触发条件**: T02-01 ~ T02-06 完成

**验收清单**:

| 检查项 | 验收标准 | 方式 |
|--------|----------|------|
| Docker Socket | Agent 容器无 `/var/run/docker.sock` 挂载 | `docker inspect` |
| Prompt 注入 | 特殊字符被转义，无命令执行漏洞 | 单元测试 |
| 配置清理 | `config.yaml`, `nodes.json` 已删除 | 文件检查 |
| 安全文档 | `docs/SECURITY.md` 已创建 | 文档存在 |

**Review 流程**:
```
1. 开发者自检 → 勾选验收清单
2. 运行安全验证脚本 → 确认通过
3. 提交 Review Request → @reviewer
4. Reviewer 检查 → 通过/打回
```

**产出物**:
- [ ] 安全修复自检报告
- [ ] 安全验证脚本输出日志

---

### CP2: 架构+存储验收 (Day 3 结束)

**触发条件**: T02-07 ~ T02-19 完成

**验收清单**:

| 检查项 | 验收标准 | 方式 |
|--------|----------|------|
| 会话目录 | `data/sessions/{session_id}/` 结构正确 | 目录检查 |
| SQLite 隔离 | 每会话独立 `.db` 文件，数据不互通 | 功能测试 |
| 路径白名单 | 非法路径访问被拒绝 | 边界测试 |
| 命令黑名单 | 危险命令被拦截 | 边界测试 |
| 文件上传 | 上传文件保存到正确会话目录 | 功能测试 |
| 文件下载 | 可下载处理结果文件 | 功能测试 |
| MinIO 移除 | docker-compose 无 MinIO 服务 | 配置检查 |

**Review 流程**:
```
1. 开发者演示会话隔离效果
2. 运行边界测试用例
3. Reviewer 检查代码实现
4. 确认架构设计文档完整
```

**产出物**:
- [ ] 架构设计文档 (`docs/architecture/session-isolation.md`)
- [ ] 边界测试报告
- [ ] 会话隔离演示录屏（可选）

## 验收标准

### Phase 2 完成标准

| 标准 | 验收方式 |
|------|----------|
| 安全止血完成 | Agent 容器无 Docker 权限，无命令注入漏洞 |
| 数据隔离生效 | 会话目录隔离，路径白名单校验通过 |
| 文件链路打通 | 上传 → 处理 → 下载全流程通过 |
| Web UI 可用 | 非技术用户可完成 Excel 处理任务 |

### 用户验收测试

```
测试场景: 文职人员处理 Excel
1. 打开浏览器访问 http://localhost:8501
2. 上传销售数据 Excel 文件
3. 输入指令: "统计每个产品的总销售额，生成汇总表"
4. 点击"执行"按钮
5. 等待处理完成
6. 下载结果文件

预期结果: 用户无需 CLI 操作，全程通过 Web 界面完成
```
